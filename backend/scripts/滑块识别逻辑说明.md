# 滑块识别逻辑说明

## 概述

滑块识别分为两个阶段：
1. **DXF 几何检测**：判定是否为滑块 + 计算角度
2. **MinIO 查表补充**：补充红色面数量和面积数据

---

## 阶段1：DXF 几何检测（slider_calculator）

**位置**：`feature_recognition/slider_calculator.py`

**触发时机**：`analyze_dxf_features` 分析 DXF 文件时

**判定逻辑**：
1. 在正视图或侧视图中检测**倾斜平行线对**（实线/虚线，颜色为红/黄/橙）
2. 平行线对长度必须 ≥ 零件长宽厚中的最小值
3. 如果检测到平行线对 → 判定为滑块

**计算内容**：
- `slider_angle`：倾斜平行线与水平/垂直线的最小角度（度）
- `code`：强制设为 `'滑块'`
- `view`：固定为 `'top_view'`（俯视图）

**输出**：
```python
{
    'code': '滑块',
    'view': 'top_view',
    'slider_angle': 15.0,  # 从几何检测得到
    'area_num': 0,         # 占位，等待查表补充
    'total_length': 0.0,   # 占位，等待查表补充
    ...
}
```

**注意**：
- 如果未检测到倾斜平行线对，`slider_angle = 0`
- 此阶段不计算面积数据

---

## 阶段2：MinIO 查表补充（apply_red_face_lookup）

**位置**：`feature_recognition/slider_red_face_lookup.py`

**触发时机**：`save_features_to_db` 保存到数据库前

**判定条件**：
- `part_name`（零件名称）包含"滑块"两字

**查表逻辑**：
1. 从 MinIO 下载 `slider/feature_database.json`
2. 用 `part_code` 查找对应零件的红色面数据
3. 补充 `area_num`（红色面数量）和 `total_length`（红色面面积和，单位 mm²）

**输出**：
```python
{
    'code': '滑块',
    'instruction': '滑块面积',  # 固定值
    'slider_angle': 15.0,       # 保留阶段1的值
    'area_num': 26,             # 从查表获取
    'total_length': 65588.829,  # 从查表获取（红色面面积和，mm²）
    'single_length': 2522.647,  # total_length / area_num
    ...
}
```

**注意**：
- 如果 `part_name` 不含"滑块"，跳过查表
- 如果查表失败，`area_num` 和 `total_length` 保持为 0

## 完整流程示例

### 示例1：零件名称含"滑块" + DXF 检测到倾斜线

**输入**：
- `part_name`: "折弯滑块夹板"
- `part_code`: "PH2-04"
- DXF 中有倾斜平行线对，角度 15°

**阶段1（DXF 检测）**：
```python
{
    'code': '滑块',
    'slider_angle': 15.0,  # ✓ 检测到
    'area_num': 0,
    'total_length': 0.0
}
```

**阶段2（查表补充）**：
```python
{
    'code': '滑块',
    'instruction': '滑块面积',
    'slider_angle': 15.0,      # ✓ 保留
    'area_num': 26,            # ✓ 查表获取
    'total_length': 65588.829  # ✓ 查表获取（红色面面积）
}
```

### 示例2：零件名称含"滑块" + DXF 未检测到倾斜线

**输入**：
- `part_name`: "下折弯滑块"
- `part_code`: "DIE-06"
- DXF 中无倾斜平行线对（或被过滤）

**阶段1（DXF 检测）**：
```python
{
    'code': 'W',  # 原始工艺编号
    'slider_angle': 0,  # ✗ 未检测到
    'area_num': 0,
    'total_length': 43.79  # 原始线割长度
}
```

**阶段2（查表补充）**：
```python
{
    'code': '滑块',            # ✓ 强制改为滑块
    'instruction': '滑块面积',
    'slider_angle': 0,         # ✗ 保持为 0（DXF 未检测到）
    'area_num': 45,            # ✓ 查表获取
    'total_length': 47395.535  # ✓ 查表获取（红色面面积）
}
```

### 示例3：零件名称不含"滑块"

**输入**：
- `part_name`: "上模座"
- `part_code`: "DIE-01"

**阶段1（DXF 检测）**：
- 正常识别线割工艺

**阶段2（查表补充）**：
- 跳过（`part_name` 不含"滑块"）

## 字段说明

| 字段 | 来源 | 单位 | 说明 |
|------|------|------|------|
| `code` | 阶段1/阶段2 | - | 工艺编号，滑块固定为 `'滑块'` |
| `instruction` | 阶段2 | - | 固定为 `'滑块面积'` |
| `slider_angle` | 阶段1 | 度 | 倾斜角度，未检测到为 0 |
| `area_num` | 阶段2 | 个 | 红色面数量 |
| `total_length` | 阶段2 | mm² | 红色面面积和 |
| `single_length` | 阶段2 | mm² | 单个红色面平均面积 |
| `view` | 阶段1 | - | 固定为 `'top_view'` |

## 常见问题

### Q1: 为什么 `slider_angle` 是 0？

**原因**：DXF 中未检测到倾斜平行线对，可能是：
1. 正视图/侧视图识别失败（板料线缺失）
2. 倾斜线颜色不在 `[1, 220, 190]`（红/黄/橙）
3. 倾斜线长度 < 零件长宽厚最小值
4. 倾斜线接近水平/垂直（角度 < 1°）

**解决**：检查 DXF 文件，确认倾斜线的颜色和长度

### Q2: 为什么 `total_length` 很大（几万）？

**正常**：`total_length` 是红色面面积和，单位是 mm²，不是线性长度

### Q3: 如何更新红色面数据？

1. 修改 `.x_t` 文件（NX 中标记红色面）
2. 重新运行 `build_slider_feature_db.py`
3. 在 Swagger 上传新的 JSON 到 MinIO
4. 重新跑特征识别
