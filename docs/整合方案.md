# 后端项目整合方案

## 一、项目现状分析

### 1. mold_cost_account_python (Flask - 账户管理)
- **端口**: 需要配置
- **核心功能**:
  - 用户登录认证 (JWT)
  - 密码管理
  - 工艺规则管理 (process_rules)
  - 价格项管理 (price_items)
  - 聊天会话管理 (chat_sessions)
  - 任务管理 (jobs)
- **技术栈**: Flask + PostgreSQL + bcrypt + PyJWT

### 2. mold_cost-main (FastAPI - 主业务系统)
- **端口**: 8300 (API Gateway)
- **核心功能**:
  - CAD文件处理和拆图
  - 特征识别 (MCP服务)
  - 价格计算
  - 任务编排 (RabbitMQ)
  - 报表生成
  - WebSocket实时通信
  - 价格加权服务
- **技术栈**: FastAPI + PostgreSQL + RabbitMQ + MinIO + Redis + LangChain

### 3. moldCost (FastAPI - AI交互系统)
- **端口**: 未明确
- **核心功能**:
  - InteractionAgent V2 (AI对话)
  - 意图识别和实体提取
  - 数据查询和修改
  - 审核流程管理
  - 消息持久化
- **技术栈**: FastAPI + PostgreSQL + Redis + LangChain

## 二、整合策略

### 推荐方案：以 mold_cost-main 为基础整合

**原因**:
1. FastAPI性能更好，异步支持完善
2. 已有完整的API Gateway架构
3. 包含核心业务逻辑（CAD处理、价格计算）
4. 基础设施完善（RabbitMQ、MinIO、Redis）

### 整合目标
- **统一端口**: 8300 (API Gateway)
- **统一框架**: FastAPI
- **统一数据库**: PostgreSQL (192.168.3.61:5432/mold_cost_db)
- **统一认证**: JWT (PyJWT)

### 第二步：合并认证模块

**目标**: 统一使用 PyJWT 进行认证

**需要合并的文件**:
- `mold_cost_account_python/main.py` 的登录逻辑 → `api_gateway/routers/auth.py`
- `mold_cost_account_python/app/services/database.py` → `shared/database.py`
- Flask的认证中间件 → FastAPI的依赖注入

**新增路由**:
```python
# api_gateway/routers/auth.py
POST /api/auth/login          # 用户登录
POST /api/auth/verify-token   # 验证token
POST /api/auth/change-password # 修改密码
POST /api/auth/refresh-token  # 刷新token
```

### 第三步：合并业务路由

**从 mold_cost_account_python 迁移**:
- `app/api/process_rules.py` → `api_gateway/routers/process_rules.py`
- `app/api/price_items.py` → `api_gateway/routers/price_items.py`
- `app/api/chat_sessions.py` → `api_gateway/routers/chat_sessions.py`
- `app/api/jobs.py` → 合并到 `api_gateway/routers/jobs.py`

**从 moldCost 迁移**:
- `agents/` 整个目录 → `agents/`
- `api_gateway/routers/` 中的交互相关路由 → `api_gateway/routers/chat.py`

## 五、启动方式

### 统一启动脚本

**start_all.bat** (Windows):
```batch
@echo off
echo 启动模具成本核算系统...

REM 启动API Gateway
start "API Gateway" cmd /k "cd mold_cost_backend && python -m uvicorn api_gateway.main:app --host 0.0.0.0 --port 8300 --reload"

REM 启动Worker
start "Worker" cmd /k "cd mold_cost_backend && python workers/all_tasks_worker.py"

REM 启动MCP服务
start "MCP Service" cmd /k "cd mold_cost_backend/mcp_services/cad_price_search_mcp && python server.py"

echo 所有服务已启动！
echo API Gateway: http://localhost:8300
echo API文档: http://localhost:8300/docs
pause
```

### 单一端口访问

所有API通过 **http://localhost:8300** 访问:
- 认证: `/api/auth/*`
- 任务: `/api/v1/jobs/*`
- 特征: `/api/features/*`
- 价格: `/api/pricing/*`
- 报表: `/api/v1/reports/*`
- 聊天: `/api/chat/*`
- WebSocket: `ws://localhost:8300/ws`

## 七、风险和注意事项

1. **数据库兼容性**: 确保三个项目使用的表结构一致
2. **JWT密钥**: 统一JWT_SECRET_KEY，避免token失效
3. **依赖版本**: 解决依赖冲突（如 fastapi、pydantic 版本）
4. **异步改造**: Flask的同步代码需要改为FastAPI的异步代码
5. **测试覆盖**: 整合后需要全面测试所有功能
