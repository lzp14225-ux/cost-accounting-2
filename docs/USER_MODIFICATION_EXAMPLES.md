## 📊 修改模式总结

### 常见句式结构

1. **"将X的Y改为Z"** - 标准修改句式
   - 将UP01的材质改为718
   - 将这套的价格改为6块

2. **"X改成Y"** - 简化句式
   - UP01改成慢丝
   - 材质改成S136

3. **"X全部改成Y"** - 批量修改
   - 冲头类零件改成慢丝
   - 这套全部改成中丝

4. **"X不对，改成Y"** - 纠错句式
   - 价格不对，改成6块
   - 材质不对，应该是718

5. **"把X改为Y"** - 口语化句式
   - 把单价改为0.0018
   - 把材质改为P20

### 关键词提取

#### 零件标识
- UP01, UP02, UP03...
- 冲头、镶件、导柱、导套、滑块、斜顶
- 这套、所有、全部

#### 属性字段
- 材质、材料
- 价格、单价
- 线割方式、工艺
- 尺寸、长度、宽度、厚度


#### 修改动作
- 改、改成、改为、修改、调整、设置
- 全部改、都改、统一改
- 不对、应该是



## 🎯 解析难点

###  批量修改的范围识别
```
冲头类的零件全部改成慢丝割一修一
```
需要识别：
- 筛选条件：零件类型 = 冲头
- 修改字段：线割方式
- 修改值：慢丝割一修一


### 表名和字段的推断
```
这套的45#价格不对，改成6块
```
需要推断：
- 表：price_snapshots
- 筛选条件：category = "material", sub_category = "45#"
- 修改字段：price
- 修改值：6.0


###  类型筛选的识别
```
下模板类的零件全部改成中丝割一修一
```
需要识别：
- 筛选条件：零件类型包含"下模板"
- 修改字段：wire_process, wire_process_note
- 修改值：middle_and_one, "中丝割一修一"
- 批量修改：所有匹配的零件
