# 外部调用自动报价流程文档

本文档说明外部系统如何在不做身份验证的情况下，通过 HTTP 接口完成：

```text
上传图纸 -> 自动拆图/特征识别/NC计算 -> 自动确认继续 -> 价格计算 -> 下载报价报表
```

当前项目没有单独的“一次调用直接返回报表”接口，但已有接口可以由外部脚本串联完成全流程。

## 1. 前置条件

调用前需要确认以下服务已启动并可用：

- 统一后端 FastAPI：`mold_main/backend/main.py`
- MCP 服务：`mold_main/backend/mcp_services/cad_price_search_mcp/server.py`
- Worker：`mold_main/backend/workers/all_tasks_worker.py`，或由 `main.py` 内嵌启动
- PostgreSQL、RabbitMQ、Redis、MinIO

本文档用变量表示地址：

```text
BASE_URL=http://127.0.0.1:8212
```

如果你的后端端口不同，以实际启动端口为准。

## 2. 总接口方式

如果外部系统希望只调用一个接口，可以启动新增的包装服务：

```bash
python tools\external_quote_api.py ^
  --host 0.0.0.0 ^
  --port 8300 ^
  --backend-base-url http://127.0.0.1:8212
```

`external_quote_api.py` 已内置当前默认配置：

```text
backend_username=admin
backend_password=123456
db_host=192.168.3.61
db_port=5432
db_name=mold_cost_db
db_user=postgres
db_password=lzp1314..
```

所以通常只需要这样启动：

```bash
python tools\external_quote_api.py
```

外部调用 `8300` 这个包装服务不需要任何身份验证。如果你当前旧后端 `8212` 仍然启用了 JWT，可以只在启动包装服务时配置内部 token：

```bash
python tools\external_quote_api.py ^
  --host 0.0.0.0 ^
  --port 8300 ^
  --backend-base-url http://127.0.0.1:8212 ^
  --backend-auth-token <后端内部token>
```

这个 token 只由包装服务内部调用旧后端时使用，外部系统不需要传。

也可以让包装服务启动时自动登录旧后端拿 token：

```bash
python tools\external_quote_api.py ^
  --host 0.0.0.0 ^
  --port 8300 ^
  --backend-base-url http://127.0.0.1:8212 ^
  --backend-username <后端用户名> ^
  --backend-password <后端密码>
```

如果看到旧后端返回：

```json
{"detail":"Not authenticated"}
```

说明包装服务内部调用旧后端时没有带 token，需要按上面两种方式之一重新启动包装服务。

另外，包装服务查询任务状态时会直接只读查询 `jobs` 表，避免旧后端 `/api/v1/jobs/{job_id}/status` 的用户权限校验导致 `PERMISSION_DENIED`。数据库配置默认读取 `.env`，也可以启动时显式指定：

```bash
python tools\external_quote_api.py ^
  --host 0.0.0.0 ^
  --port 8300 ^
  --backend-base-url http://127.0.0.1:8212 ^
  --backend-username <后端用户名> ^
  --backend-password <后端密码> ^
  --db-host 127.0.0.1 ^
  --db-port 5432 ^
  --db-name mold_cost_db ^
  --db-user postgres ^
  --db-password <数据库密码>
```

外部只需要调用：

```http
POST /api/external/quote
Content-Type: multipart/form-data
```

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `dwg_file` | DWG/PRT 至少一个 | DWG 图纸文件 |
| `prt_file` | DWG/PRT 至少一个 | PRT 三维文件 |
| `timeout_seconds` | 否 | 最大等待秒数，默认 `7200` |
| `poll_interval` | 否 | 轮询间隔秒数，默认 `5` |
| `backend_base_url` | 否 | 后端地址，默认启动参数里的 `--backend-base-url` |

curl 示例：

```bash
curl -L -X POST "http://127.0.0.1:8300/api/external/quote" ^
  -F "dwg_file=@D:\files\demo.dwg" ^
  -F "prt_file=@D:\files\demo.prt" ^
  -F "timeout_seconds=7200" ^
  -F "poll_interval=5" ^
  -o "D:\reports\demo_报价单.xlsx"
```

这个总接口内部会自动完成：

```text
上传图纸 -> 查询状态 -> awaiting_confirm 自动 continue -> completed 后下载报表 -> 把 xlsx 返回给外部调用方
```

响应头里会带：

```http
X-Job-Id: <job_id>
Content-Disposition: attachment; filename="..."
```

## 3. 直接调用现有接口的流程

```text
1. 上传 DWG/PRT，创建任务，拿到 job_id
2. 轮询任务状态
3. 当状态变成 awaiting_confirm 时，自动调用 continue
4. 继续轮询，直到状态变成 completed
5. 调用报表导出接口，保存 xlsx 文件
```

关键状态：

| status | 含义 | 外部脚本动作 |
| --- | --- | --- |
| `pending` | 任务已创建，等待处理 | 继续轮询 |
| `processing` | 后台处理中 | 继续轮询 |
| `awaiting_confirm` | 等待人工确认 | 外部脚本自动调用 continue |
| `completed` | 全流程完成 | 下载报表 |
| `failed` | 任务失败 | 停止并输出错误 |

## 4. 上传图纸创建任务

接口：

```http
POST /api/v1/jobs/upload
Content-Type: multipart/form-data
```

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `dwg_file` | DWG/PRT 至少一个 | DWG 图纸文件 |
| `prt_file` | DWG/PRT 至少一个 | PRT 三维文件，上传后可参与 NC 时间计算 |
| `encryption_key` | 否 | 当前预留 |

curl 示例：

```bash
curl -X POST "%BASE_URL%/api/v1/jobs/upload" ^
  -F "dwg_file=@D:\files\demo.dwg" ^
  -F "prt_file=@D:\files\demo.prt"
```

响应示例：

```json
{
  "job_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "status": "pending",
  "message": "文件上传成功，任务已创建，正在处理...",
  "files": {
    "dwg": {
      "filename": "demo.dwg",
      "size": 123456
    },
    "prt": {
      "filename": "demo.prt",
      "size": 987654
    }
  }
}
```

## 6. 自动确认继续核算

现有主流程在特征识别和 NC 计算后会进入人工确认状态：

```text
status=awaiting_confirm
current_stage=awaiting_confirm
```

外部脚本如果不需要人工审核，可以在第一次检测到该状态时自动调用：

```http
POST /api/v1/jobs/{job_id}/continue
```

curl 示例：

```bash
curl -X POST "%BASE_URL%/api/v1/jobs/a1b2c3d4-e5f6-7890-abcd-ef1234567890/continue"
```

响应示例：

```json
{
  "status": "accepted",
  "message": "任务已开始继续核算，请通过 WebSocket 查看进度",
  "job_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

注意：脚本应避免重复调用 continue。建议在客户端保存一个布尔变量：

```text
continued=false
```

只有第一次看到 `awaiting_confirm` 时调用 continue，然后设置为 `continued=true`。

## 7. 下载报价报表

当任务状态变成：

```text
status=completed
```

即可下载报表。

接口：

```http
GET /api/v1/reports/{job_id}/export?format=xlsx&async_mode=false
```

curl 示例：

```bash
curl -L -X GET "%BASE_URL%/api/v1/reports/a1b2c3d4-e5f6-7890-abcd-ef1234567890/export?format=xlsx&async_mode=false" ^
  -o "D:\reports\demo_报价单.xlsx"
```

说明：

- 当前报表接口会返回 `.xlsx` 文件流。
- `async_mode=false` 表示同步生成并直接下载。
- 如果任务子图数量非常大，可以使用异步导出模式，但外部自动脚本通常建议先使用同步下载。

## 8. 外部脚本伪代码

```python
job_id = upload_files(
    base_url=base_url,
    dwg_path=dwg_path,
    prt_path=prt_path,
)

continued = False

while True:
    status_data = get_job_status(base_url, job_id)
    status = status_data["status"]

    if status == "awaiting_confirm" and not continued:
        continue_job(base_url, job_id)
        continued = True

    elif status == "completed":
        break

    elif status == "failed":
        raise RuntimeError(status_data)

    sleep(5)

download_report(base_url, job_id, output_path)
```

## 9. Python 客户端示例

下面是最小可用示例，可作为外部脚本的实现参考。

```python
import argparse
import time
from pathlib import Path

import requests


def upload_job(base_url: str, dwg_path: str | None, prt_path: str | None) -> str:
    files = {}

    try:
        if dwg_path:
            files["dwg_file"] = (Path(dwg_path).name, open(dwg_path, "rb"))
        if prt_path:
            files["prt_file"] = (Path(prt_path).name, open(prt_path, "rb"))

        resp = requests.post(
            f"{base_url}/api/v1/jobs/upload",
            files=files,
            timeout=300,
        )
        resp.raise_for_status()
        return resp.json()["job_id"]
    finally:
        for _, file_obj in files.values():
            file_obj.close()


def get_status(base_url: str, job_id: str) -> dict:
    resp = requests.get(
        f"{base_url}/api/v1/jobs/{job_id}/status",
        timeout=30,
    )
    resp.raise_for_status()
    return resp.json()


def continue_job(base_url: str, job_id: str) -> None:
    resp = requests.post(
        f"{base_url}/api/v1/jobs/{job_id}/continue",
        timeout=30,
    )
    resp.raise_for_status()


def wait_completed(base_url: str, job_id: str, timeout_seconds: int, poll_interval: int) -> None:
    deadline = time.time() + timeout_seconds
    continued = False

    while time.time() < deadline:
        data = get_status(base_url, job_id)
        status = data.get("status")
        stage = data.get("current_stage")
        progress = data.get("progress")
        print(f"job={job_id} status={status} stage={stage} progress={progress}")

        if status == "awaiting_confirm" and not continued:
            continue_job(base_url, job_id)
            continued = True
            print("auto continue submitted")

        elif status == "completed":
            return

        elif status == "failed":
            raise RuntimeError(f"job failed: {data}")

        time.sleep(poll_interval)

    raise TimeoutError(f"job timeout: {job_id}")


def download_report(base_url: str, job_id: str, output_path: str) -> None:
    resp = requests.get(
        f"{base_url}/api/v1/reports/{job_id}/export",
        params={"format": "xlsx", "async_mode": "false"},
        timeout=600,
    )
    resp.raise_for_status()

    output = Path(output_path)
    output.parent.mkdir(parents=True, exist_ok=True)
    output.write_bytes(resp.content)


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--base-url", required=True)
    parser.add_argument("--dwg")
    parser.add_argument("--prt")
    parser.add_argument("--output", required=True)
    parser.add_argument("--timeout", type=int, default=7200)
    parser.add_argument("--poll-interval", type=int, default=5)
    args = parser.parse_args()

    if not args.dwg and not args.prt:
        raise SystemExit("dwg or prt is required")

    job_id = upload_job(args.base_url, args.dwg, args.prt)
    print(f"job_id={job_id}")

    wait_completed(args.base_url, job_id, args.timeout, args.poll_interval)
    download_report(args.base_url, job_id, args.output)
    print(f"report saved: {args.output}")


if __name__ == "__main__":
    main()
```

运行示例：

```bash

  --base-url http://127.0.0.1:8212 ^
  --dwg D:\files\demo.dwg ^
  --prt D:\files\demo.prt ^

```

## 10. 常见问题

### 10.1 上传后一直 processing

优先检查：

- Worker 是否启动
- RabbitMQ 是否正常
- MCP 服务是否启动
- `mold_main/backend/logs/all_tasks_worker.log`
- `mold_main/backend/logs/mcp_service.log`
- `mold_main/backend/logs/scripts.log`

### 10.2 一直停在 awaiting_confirm

说明脚本没有成功调用：

```http
POST /api/v1/jobs/{job_id}/continue
```

检查 continue 请求是否返回 `status=accepted`。

### 10.3 报表导出失败

确认任务已经是：

```text
status=completed
```

如果还没完成就导出，可能出现缺少子图数据、缺少价格数据或报表为空。

### 10.4 是否支持一个 HTTP 请求直接等到报表返回

已经支持，使用 `tools/external_quote_api.py` 提供的：

```http
POST /api/external/quote
```

但需要注意：CAD 拆图、特征识别、NC 计算和价格计算耗时可能较长，外部调用方的 HTTP 超时时间要设置得足够长。更稳的集成方式仍然是：

```text
上传 -> 获取 job_id -> 轮询状态 -> 下载报表
```
