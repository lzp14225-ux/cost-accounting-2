# 外部调用自动报价流程文档

本文档说明如何在不修改现有项目代码的前提下，让外部系统通过 HTTP 接口完成：

```text
上传图纸 -> 自动拆图/特征识别/NC计算 -> 自动确认继续 -> 价格计算 -> 下载报价报表
```

当前项目没有单独的“一次调用直接返回报表”接口，但已有接口可以由外部脚本串联完成全流程。

## 1. 前置条件

调用前需要确认以下服务已启动并可用：

- 统一后端 FastAPI：`mold_main/backend/main.py`
- MCP 服务：`mold_main/backend/mcp_services/cad_price_search_mcp/server.py`
- Worker：`mold_main/backend/workers/all_tasks_worker.py`，或由 `main.py` 内嵌启动
- PostgreSQL、RabbitMQ、Redis、MinIO

本文档用变量表示地址：

```text
BASE_URL=http://127.0.0.1:8212
```

如果你的后端端口不同，以实际启动端口为准。

## 2. 总体调用流程

```text
1. 登录获取 JWT token
2. 上传 DWG/PRT，创建任务，拿到 job_id
3. 轮询任务状态
4. 当状态变成 awaiting_confirm 时，自动调用 continue
5. 继续轮询，直到状态变成 completed
6. 调用报表导出接口，保存 xlsx 文件
```

关键状态：

| status | 含义 | 外部脚本动作 |
| --- | --- | --- |
| `pending` | 任务已创建，等待处理 | 继续轮询 |
| `processing` | 后台处理中 | 继续轮询 |
| `awaiting_confirm` | 等待人工确认 | 外部脚本自动调用 continue |
| `completed` | 全流程完成 | 下载报表 |
| `failed` | 任务失败 | 停止并输出错误 |

## 3. 登录获取 Token

接口：

```http
POST /api/login
Content-Type: application/json
```

请求示例：

```bash
curl -X POST "%BASE_URL%/api/login" ^
  -H "Content-Type: application/json" ^
  -d "{\"username\":\"YOUR_USERNAME\",\"password\":\"YOUR_PASSWORD\"}"
```

响应示例：

```json
{
  "success": true,
  "message": "登录成功",
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user_info": {
    "user_id": "xxx",
    "username": "demo"
  }
}
```

后续需要在请求头里带上：

```http
Authorization: Bearer <token>
```

## 4. 上传图纸创建任务

接口：

```http
POST /api/v1/jobs/upload
Authorization: Bearer <token>
Content-Type: multipart/form-data
```

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `dwg_file` | DWG/PRT 至少一个 | DWG 图纸文件 |
| `prt_file` | DWG/PRT 至少一个 | PRT 三维文件，上传后可参与 NC 时间计算 |
| `encryption_key` | 否 | 当前预留 |

curl 示例：

```bash
curl -X POST "%BASE_URL%/api/v1/jobs/upload" ^
  -H "Authorization: Bearer <token>" ^
  -F "dwg_file=@D:\files\demo.dwg" ^
  -F "prt_file=@D:\files\demo.prt"
```

响应示例：

```json
{
  "job_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "status": "pending",
  "message": "文件上传成功，任务已创建，正在处理...",
  "files": {
    "dwg": {
      "filename": "demo.dwg",
      "size": 123456
    },
    "prt": {
      "filename": "demo.prt",
      "size": 987654
    }
  }
}
```

## 6. 自动确认继续核算

现有主流程在特征识别和 NC 计算后会进入人工确认状态：

```text
status=awaiting_confirm
current_stage=awaiting_confirm
```

外部脚本如果不需要人工审核，可以在第一次检测到该状态时自动调用：

```http
POST /api/v1/jobs/{job_id}/continue
Authorization: Bearer <token>
```

curl 示例：

```bash
curl -X POST "%BASE_URL%/api/v1/jobs/a1b2c3d4-e5f6-7890-abcd-ef1234567890/continue" ^
  -H "Authorization: Bearer <token>"
```

响应示例：

```json
{
  "status": "accepted",
  "message": "任务已开始继续核算，请通过 WebSocket 查看进度",
  "job_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}
```

注意：脚本应避免重复调用 continue。建议在客户端保存一个布尔变量：

```text
continued=false
```

只有第一次看到 `awaiting_confirm` 时调用 continue，然后设置为 `continued=true`。

## 7. 下载报价报表

当任务状态变成：

```text
status=completed
```

即可下载报表。

接口：

```http
GET /api/v1/reports/{job_id}/export?format=xlsx&async_mode=false
```

curl 示例：

```bash
curl -L -X GET "%BASE_URL%/api/v1/reports/a1b2c3d4-e5f6-7890-abcd-ef1234567890/export?format=xlsx&async_mode=false" ^
  -H "Authorization: Bearer <token>" ^
  -o "D:\reports\demo_报价单.xlsx"
```

说明：

- 当前报表接口会返回 `.xlsx` 文件流。
- `async_mode=false` 表示同步生成并直接下载。
- 如果任务子图数量非常大，可以使用异步导出模式，但外部自动脚本通常建议先使用同步下载。

## 8. 外部脚本伪代码

```python
token = login(base_url, username, password)

job_id = upload_files(
    base_url=base_url,
    token=token,
    dwg_path=dwg_path,
    prt_path=prt_path,
)

continued = False

while True:
    status_data = get_job_status(base_url, token, job_id)
    status = status_data["status"]

    if status == "awaiting_confirm" and not continued:
        continue_job(base_url, token, job_id)
        continued = True

    elif status == "completed":
        break

    elif status == "failed":
        raise RuntimeError(status_data)

    sleep(5)

download_report(base_url, token, job_id, output_path)
```

## 9. Python 客户端示例

下面是最小可用示例，可作为外部脚本的实现参考。

```python
import argparse
import time
from pathlib import Path

import requests


def login(base_url: str, username: str, password: str) -> str:
    resp = requests.post(
        f"{base_url}/api/login",
        json={"username": username, "password": password},
        timeout=30,
    )
    resp.raise_for_status()
    data = resp.json()
    if not data.get("success"):
        raise RuntimeError(f"login failed: {data}")
    return data["token"]


def upload_job(base_url: str, token: str, dwg_path: str | None, prt_path: str | None) -> str:
    headers = {"Authorization": f"Bearer {token}"}
    files = {}

    try:
        if dwg_path:
            files["dwg_file"] = (Path(dwg_path).name, open(dwg_path, "rb"))
        if prt_path:
            files["prt_file"] = (Path(prt_path).name, open(prt_path, "rb"))

        resp = requests.post(
            f"{base_url}/api/v1/jobs/upload",
            headers=headers,
            files=files,
            timeout=300,
        )
        resp.raise_for_status()
        return resp.json()["job_id"]
    finally:
        for _, file_obj in files.values():
            file_obj.close()


def get_status(base_url: str, token: str, job_id: str) -> dict:
    resp = requests.get(
        f"{base_url}/api/v1/jobs/{job_id}/status",
        headers={"Authorization": f"Bearer {token}"},
        timeout=30,
    )
    resp.raise_for_status()
    return resp.json()


def continue_job(base_url: str, token: str, job_id: str) -> None:
    resp = requests.post(
        f"{base_url}/api/v1/jobs/{job_id}/continue",
        headers={"Authorization": f"Bearer {token}"},
        timeout=30,
    )
    resp.raise_for_status()


def wait_completed(base_url: str, token: str, job_id: str, timeout_seconds: int, poll_interval: int) -> None:
    deadline = time.time() + timeout_seconds
    continued = False

    while time.time() < deadline:
        data = get_status(base_url, token, job_id)
        status = data.get("status")
        stage = data.get("current_stage")
        progress = data.get("progress")
        print(f"job={job_id} status={status} stage={stage} progress={progress}")

        if status == "awaiting_confirm" and not continued:
            continue_job(base_url, token, job_id)
            continued = True
            print("auto continue submitted")

        elif status == "completed":
            return

        elif status == "failed":
            raise RuntimeError(f"job failed: {data}")

        time.sleep(poll_interval)

    raise TimeoutError(f"job timeout: {job_id}")


def download_report(base_url: str, token: str, job_id: str, output_path: str) -> None:
    resp = requests.get(
        f"{base_url}/api/v1/reports/{job_id}/export",
        params={"format": "xlsx", "async_mode": "false"},
        headers={"Authorization": f"Bearer {token}"},
        timeout=600,
    )
    resp.raise_for_status()

    output = Path(output_path)
    output.parent.mkdir(parents=True, exist_ok=True)
    output.write_bytes(resp.content)


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--base-url", required=True)
    parser.add_argument("--username", required=True)
    parser.add_argument("--password", required=True)
    parser.add_argument("--dwg")
    parser.add_argument("--prt")
    parser.add_argument("--output", required=True)
    parser.add_argument("--timeout", type=int, default=7200)
    parser.add_argument("--poll-interval", type=int, default=5)
    args = parser.parse_args()

    if not args.dwg and not args.prt:
        raise SystemExit("dwg or prt is required")

    token = login(args.base_url, args.username, args.password)
    job_id = upload_job(args.base_url, token, args.dwg, args.prt)
    print(f"job_id={job_id}")

    wait_completed(args.base_url, token, job_id, args.timeout, args.poll_interval)
    download_report(args.base_url, token, job_id, args.output)
    print(f"report saved: {args.output}")


if __name__ == "__main__":
    main()
```

运行示例：

```bash
python auto_quote_client.py ^
  --base-url http://127.0.0.1:8212 ^
  --username demo ^
  --password 123456 ^
  --dwg D:\files\demo.dwg ^
  --prt D:\files\demo.prt ^
  --output D:\reports\demo_报价单.xlsx
```

## 10. 常见问题

### 10.1 上传后一直 processing

优先检查：

- Worker 是否启动
- RabbitMQ 是否正常
- MCP 服务是否启动
- `mold_main/backend/logs/all_tasks_worker.log`
- `mold_main/backend/logs/mcp_service.log`
- `mold_main/backend/logs/scripts.log`

### 10.2 一直停在 awaiting_confirm

说明脚本没有成功调用：

```http
POST /api/v1/jobs/{job_id}/continue
```

检查 token 是否有效，以及 continue 请求是否返回 `status=accepted`。

### 10.3 报表导出失败

确认任务已经是：

```text
status=completed
```

如果还没完成就导出，可能出现缺少子图数据、缺少价格数据或报表为空。

### 10.4 是否支持一个 HTTP 请求直接等到报表返回

不建议。CAD 拆图、特征识别、NC 计算和价格计算耗时可能较长，一个同步 HTTP 请求容易超时。

推荐外部系统使用：

```text
上传 -> 获取 job_id -> 轮询状态 -> 下载报表
```

如果必须“一次请求返回报表”，建议另写一个独立包装服务，由包装服务内部执行本文档中的轮询逻辑，不建议直接改现有主后端流程。
